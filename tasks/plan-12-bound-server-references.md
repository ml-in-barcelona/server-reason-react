# Bound server references (partial application / `.bind()`)

**Priority**: Medium
**Origin**: Phase D from plan-03-rethink-server-actions-architecture.md (Layer 1.1, Layer 2.1)

## Problem

Server functions currently cannot be partially applied. React's model supports `.bind()` on server references to pre-fill arguments — the bound args are serialized as `{ id, bound: [...] }` in the RSC model stream (`$F`). On the server, `decodeReply` resolves `$F` by looking up the function and calling `.bind.apply(fn, [null].concat(boundArgs))`.

**Use case**: Passing a server action with pre-filled context to a client component:
```reason
// Server component
[@react.component]
let make = (~noteId: string) => {
  let note = Database.getNote(noteId);
  let boundDelete = deleteNote.call(noteId);
  <DeleteButton action={boundDelete} />
};
```

## Core Insight: Currying = `.bind()`

OCaml/Reason functions are curried (arity 1). Partial application IS binding:

| OCaml/Reason | JavaScript equivalent |
|---|---|
| `foo(x)` | `foo.bind(null, x)` |
| `foo(x)(y)` | `foo.bind(null, x, y)` |

However, **refmt normalizes currying syntax**: `(a) => (b) => body` becomes `(a, b) => body`. The currying structure cannot be used as the binding boundary.

### Solution: `[@react.server.function]` as a divider attribute

Place the attribute on the inner function expression. Everything before it is "bound", everything after is "call":

```reason
// Form 1 — on let binding (current, no binding)
[@react.server.function]
let action = (~content: string): Js.Promise.t(string) => body;

// Form 2 — on inner expression (new, with binding)
let deleteNote = (~noteId: string) =>
  [@react.server.function] ((~content: string): Js.Promise.t(string) => body);
```

The attribute prevents refmt from flattening. Verified:

```reason
// Input
let foo = (~a: string, ~b: int) => [@react.server.function] ((~c: string): string => c);

// refmt output (attribute preserved, structure intact)
let foo = (~a: string, ~b: int) =>
  [@react.server.function] ((~c: string) => (c: string));

// OCaml AST
let foo ~a:(a : string) ~b:(b : int) =
  ((fun ~c:(c : string) -> (c : string))
  [@react.server.function ])
```

Multiple bound params flatten together (fine — they're all bound):
```reason
let foo = (~a: string, ~b: int) =>
  [@react.server.function] ((~c: string, ~d: bool): Js.Promise.t(string) => body);
// a, b = bound; c, d = call
```

## Design: Closure-based bound encoding

### `Runtime.server_function` type change

```ocaml
(* Runtime.mli *)
type 'callback server_function = {
  id : string;
  call : 'callback;
  encode_bound : unit -> string option;
  (** Returns a JSON array string of bound args (e.g. ["abc", 42]),
      or None for unbound server functions.
      The closure captures the actual bound values and their encoders. *)
}

val make : id:string -> call:'callback -> 'callback server_function
(** Creates an unbound server function (encode_bound returns None) *)
```

```ocaml
(* Runtime.ml *)
type 'callback server_function = {
  id : string;
  call : 'callback;
  encode_bound : unit -> string option;
}

let make ~id ~call = { id; call; encode_bound = (fun () -> None) }
```

**Why a closure?**
- The closure captures bound values AND their type-specific encoders (generated by the PPX which has type info)
- Runtime stays cross-platform — `string option` works on both native and Melange
- No `Yojson.Basic.t` dependency in Runtime
- The Melange side always uses `(fun () -> None)` — bound encoding is a server-side concern
- Follows OCaml idiom: use closures to carry state + behavior together

### Type structure

Each currying level wraps the return in `server_function(...)`:

```
// Flat — no binding (same as today)
[@react.server.function]
let action = (~a: string, ~b: int): Js.Promise.t(string) => { ... };
// type: server_function((~a:string, ~b:int) -> Js.Promise.t(string))

// With binding
let action = (~a: string) =>
  [@react.server.function] ((~b: int): Js.Promise.t(string) => { ... });
// type: server_function(string -> server_function(int -> Js.Promise.t(string)))
//
// action.call("x")          : server_function(int -> Js.Promise.t(string))  ← serializable
// action.call("x").call(42) : Js.Promise.t(string)                         ← final call
```

### Native (server) output

```reason
// Source:
let deleteNote = (~noteId: string) =>
  [@react.server.function] ((~content: string): Js.Promise.t(string) => {
    Database.delete(noteId, content)
  });

// Generated (native):
include {
  // 1. Raw body — used by FunctionReferences (avoids intermediate allocations)
  let deleteNote__body = (~noteId: string, ~content: string): Js.Promise.t(string) => {
    Database.delete(noteId, content)
  };

  // 2. Public API — bound params produce a new server_function
  let deleteNote = {
    Runtime.id: "<hash>",
    encode_bound: () => None,
    call: (~noteId: string) => {
      {
        Runtime.id: "<hash>",
        encode_bound: () => Some(
          Printf.sprintf("[%s]",
            Yojson.Basic.to_string(
              Melange_json.Primitives.string_to_json(noteId)))
        ),
        call: (~content: string): Js.Promise.t(string) => {
          deleteNote__body(~noteId, ~content)
        }
      }
    }
  };

  // 3. Registration — flat, receives ALL args (bound ++ call)
  FunctionReferences.register("<hash>", Body(fun args ->
    let noteId = string_of_json(args.(0)) in
    let content = string_of_json(args.(1)) in
    deleteNote__body(~noteId, ~content) |> Lwt.map(to_rsc)
  ));
};
```

### Melange (client) output

## RSC Serialization

### `action_to_json` change (`ReactServerDOM.ml:197`)

```ocaml
(* Before *)
let action_to_json (action : _ Runtime.server_function) =
  `Assoc [ ("id", `String action.id); ("bound", `Null) ]

(* After *)
let action_to_json (action : _ Runtime.server_function) =
  let bound = match action.encode_bound () with
    | None -> `Null
    | Some json_str -> Yojson.Basic.from_string json_str
  in
  `Assoc [ ("id", `String action.id); ("bound", bound) ]
```

## Server Dispatch — No Changes Needed

The server handler registered in `FunctionReferences` expects ALL args in a flat `Yojson.Basic.t array`. When a bound action is invoked from the client:

1. Client has `$$bound = [noteId_json]` from the RSC stream
2. Client calls with `(content)` → React prepends bound args
3. `callServer(id, encodeReply([noteId, content]))` → server receives flat array
4. Handler was registered with all params → works as-is

**No change to `FunctionReferences`, `decodeReply`, or `DreamRSC` dispatch.**

## Open Questions

### Labeled arguments in bound levels

Labels are lost when serialized as JSON. The server-side handler uses positional indexing (`args.(0)`, `args.(1)`) regardless of labels. This should work because:
- Bound args are always prepended (positions 0..n-1)
- Call args follow (positions n..m)
- The handler decodes by position, matching the original parameter order

### Optional arguments with defaults

```reason
let foo = (~name: string="Lola") =>
  [@react.server.function] ((~content: string): Js.Promise.t(string) => body);
```

When `foo.call()` is called (using the default), `name` resolves to `"Lola"`. The bound arg closure captures the resolved value. The server handler receives `"Lola"` as `args.(0)`. This should work correctly.

### FormData in bound levels

A `Js.FormData.t` parameter doesn't make sense in bound position (FormData comes from the client at call time). The PPX should error:

```reason
// ERROR: Js.FormData.t cannot be a bound parameter
let foo = (formData: Js.FormData.t) =>
  [@react.server.function] ((~content: string): Js.Promise.t(string) => body);
```

### Progressive enhancement for bound actions

For `<form action={boundAction}>`, the server currently renders `$ACTION_ID_<hash>` hidden inputs. Bound actions would need a different protocol — `$ACTION_REF_<prefix>` with bound args serialized as additional FormData entries. This is a separate concern (connects to plan-10-encode-form-action).

### `make_json_encoder` for bound args

The PPX currently has `make_json_decoder` (for deserializing args on the server) but needs the inverse `make_json_encoder` (for serializing bound args into JSON). The encoder maps OCaml types to `Yojson.Basic.to_string` calls:

| Type | Encoder |
|---|---|
| `string` | `` Yojson.Basic.to_string (`String value) `` |
| `int` | `` Yojson.Basic.to_string (`Int value) `` |
| `float` | `` Yojson.Basic.to_string (`Float value) `` |
| `bool` | `` Yojson.Basic.to_string (`Bool value) `` |
| `'a option` | Encode inner or `null` |
| `'a list` | Encode as JSON array |
| Custom types | Use `[%to_json: type]` (Melange_json) |
